# Sanmao 轻量真实版设计文档

**目标**

将当前纯前端演示站升级为运行在远程服务器上的轻量真实版产品。新增后端和 SQLite 数据持久化，使用户名注册、资料补全、喜欢关系、匹配关系和聊天消息都能真实保存。

**核心流程重排**

- 首次进入只要求填写用户名。
- 用户名需要做唯一性校验。
- 用户注册后即可浏览资料和点喜欢。
- 当用户第一次尝试发消息时，如果资料未完善，则强制进入完整资料补全流程。
- 完整资料补全后，聊天功能解锁。

**产品结构**

- `发现`：只展示刷卡和每日浏览配额。
- `喜欢`：只展示 `我喜欢的人` 和 `喜欢我的人`。
- `消息`：只展示匹配会话和聊天内容。
- `我的`：展示个人资料并支持编辑。

**后端方案**

- 技术选型：Python 标准库 HTTP 服务 + SQLite
- 数据文件部署在远程服务器，例如 `/var/www/sanmao/data/sanmao.db`
- 前端通过 JSON API 与后端交互

**数据模型**

- `users`
- `profiles`
- `likes`
- `matches`
- `messages`

**真实逻辑**

- 用户名注册时查重。
- 发现页候选人从数据库读取。
- 喜欢关系写库。
- 双向喜欢自动生成匹配。
- 消息写库并按时间排序。
- 资料完整度由数据库字段控制。

**验证方式**

- 单元测试：用户名校验、匹配生成、消息追加、资料补全门禁。
- 手动验证：注册、浏览、喜欢、被喜欢、匹配、补全资料、发送消息。
