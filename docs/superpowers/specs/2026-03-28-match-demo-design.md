# 深圳年轻人相亲演示站设计文档

**目标**

构建一个可直接部署到 Vercel 的相亲产品演示站，面向“深圳大厂刚毕业的小年轻”这一人群。产品以移动端刷卡体验为核心，用假数据完成最小闭环：进入应用、填写资料、每日查看异性资料、右滑喜欢、双向匹配、进入假聊天。

**产品定位**

这不是传统婚恋站，也不是单页海报站，而是一个偏年轻、克制、精致的相亲产品原型。视觉上参考“二狗 / 青藤”这类真实感较强、不过度营销的产品表达。

**范围**

- 首页/引导页：介绍产品价值和演示规则。
- 资料填写页：输入昵称、年龄、职业、公司、学校、城市、个人标签与自我介绍。
- 刷卡主页：一次只展示一位异性资料，支持喜欢和跳过。
- 每日额度：每日最多查看 30 份异性资料。
- 匹配逻辑：对预设命中用户点击喜欢时，触发双向匹配成功。
- 假聊天页：匹配后进入演示聊天界面，消息为预设内容。

**非目标**

- 不接真实数据库。
- 不做真实账号系统。
- 不做多人同步聊天。
- 不做复杂筛选、付费、风控与审核。

**数据与状态**

- 假数据内置在前端脚本中。
- 本地状态使用浏览器 `localStorage` 持久化。
- 记录字段包括：当前资料、今日查看计数、已喜欢用户、已匹配用户、资料表单内容。
- 每日 30 次配额基于本地日期重置。

**交互规则**

- 用户首次进入需完成资料填写才能开始刷卡。
- 刷卡只展示与当前用户性别相反的资料。
- 每看过一张资料即计入当日已查看数量。
- 达到 30 张后显示“今日已看完”状态，不再继续发卡。
- 对预设为“喜欢当前用户”的资料右滑时，弹出匹配成功层并可进入聊天。

**视觉方向**

- 以移动端单手操作为中心。
- 色彩偏奶白、墨黑、鼠尾草绿或低饱和暖色，不走俗艳粉紫路线。
- 大面积留白、卡片感照片区、真实标签与轻微动效。

**技术方案**

- 纯静态站：HTML + CSS + 原生 JavaScript 模块。
- 使用 Node 内置测试运行核心业务逻辑测试。
- Vercel 以静态站方式部署。

**验证方式**

- 核心逻辑单元测试：每日 30 次限制、候选人筛选、喜欢与匹配逻辑、聊天线程读取。
- 手动验证：资料填写、刷卡、额度耗尽、匹配弹层、聊天页展示、刷新后状态保留。
