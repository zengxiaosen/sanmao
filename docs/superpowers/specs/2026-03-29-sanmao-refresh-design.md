# Sanmao 第二版体验升级设计文档

**目标**

在保留当前假数据演示闭环的前提下，把 `Sanmao` 从“能跑的演示页”升级成“更像真实年轻人社交产品”的单页应用。重点提升视觉完成度、信息层级和产品感，而不是增加真实后端。

**升级定位**

新版气质定义为“年轻松弛”。用户感知应从“这是一个 demo”转向“这是一个已经有方向、有审美、有产品结构的早期产品”。视觉表达要更轻盈、生活化、可呼吸，不做婚恋站土味表达。

**范围**

- 重构首页头部，让产品介绍更像真实应用入口。
- 刷卡区升级为主卡片 + 资料详情抽屉。
- 新增“我喜欢的人”列表，展示已右滑对象。
- 升级“已匹配”区域，让其更像聊天产品中的会话列表。
- 升级每日浏览配额展示，加入进度表达和状态提示。
- 优化匹配弹层和聊天界面层次。
- 强化空状态、引导文案和微动效。

**非目标**

- 不接数据库。
- 不做真实注册登录。
- 不做真实聊天发送。
- 不做复杂筛选器和推荐算法。

**交互设计**

- 刷卡主卡片保留快速决策能力。
- 增加“查看完整资料”入口，打开侧边详情抽屉。
- 右滑后用户进入“我喜欢的人”，命中互相喜欢则同时进入“已匹配”。
- 聊天区不再仅作为弹层入口，而是页面内的稳定模块。
- 每日配额使用文本 + 进度条双重表达，让“今日可看 30 个”的规则更清晰。

**信息结构**

- 左侧：产品引导、今日配额、刷卡主卡片。
- 中部或附属层：完整资料详情抽屉。
- 右侧：我喜欢的人、已匹配会话。
- 顶部文案强调“认真、轻量、同城、年轻职场人”。

**视觉方向**

- 配色继续使用奶白、暖黑、鼠尾草绿，但加入更柔和的层次和更细腻的阴影。
- 版式从“单卡片展示”升级为“有节奏的产品面板拼接”。
- 卡片、抽屉、匹配列表、聊天窗统一视觉语言。
- 动效以轻微位移、透明度、层叠感为主，不做夸张炫技。

**技术方案**

- 保持纯静态站架构。
- 将当前较大的 `src/app.js` 按职责拆分为状态辅助与界面渲染辅助。
- 在核心逻辑层增加面向新版 UI 的纯函数，保障“当前卡片详情”“喜欢列表”“匹配列表”来源稳定可测。

**验证方式**

- 单元测试覆盖：
  - 当前卡片详情读取
  - 喜欢列表映射
  - 已匹配聊天线程读取
  - 每日额度逻辑不回归
- 手动验证覆盖：
  - 资料填写
  - 查看完整资料
  - 喜欢/跳过
  - 喜欢列表更新
  - 匹配弹层出现
  - 聊天区展示
  - 刷新后状态保留
